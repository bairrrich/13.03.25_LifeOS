# Product Context

## Why This Exists
Современные инструменты продуктивности фрагментированы: задачи в одном месте, цели в другом, привычки в третьем. LifeOS объединяет всё в единой системе с целостным подходом к управлению личной эффективностью.

## Problems It Solves
| Problem | Impact | Solution |
|---------|--------|----------|
| Фрагментация инструментов | Потеря контекста, дублирование | Единая система для всех аспектов |
| Отсутствие связи задач и целей | Тактика без стратегии | Иерархия: цели → проекты → задачи |
| Сложность отслеживания прогресса | Демотивация | Визуализация прогресса по всем уровням |
| Контекст теряется между сессиями | Время на восстановление | Memory Bank для AI-агентов |

## User Experience Goals
1. **Clarity**: Ясное понимание что делать прямо сейчас
2. **Focus**: Минимизация когнитивной нагрузки
3. **Progress**: Видимый прогресс по целям
4. **Flow**: Бесшовный workflow между планированием и выполнением

## Target User
- Разработчик/инженер (технический склад ума)
- Использует Qwen Code для разработки
- Ценит spec-driven подход
- Предпочитает Markdown/текстовые форматы

## Competitive Landscape
| Tool | Strength | Weakness | LifeOS Differentiator |
|------|----------|----------|----------------------|
| Notion | Гибкость | Требует настройки | Готовая структура |
| Obsidian | Локальность | Нет встроенного трекинга | Spec-driven + AI |
| Todoist | Простота | Только задачи | Целостная система |

## Success Criteria
- Ежедневное использование для планирования
- Все личные цели задокументированы и отслеживаются
- AI-агент может продолжить работу без потерь контекста
