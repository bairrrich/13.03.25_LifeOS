# /speckit.clarify

## Purpose
Clarify and refine the specification.

## Instructions
1. Review existing specification
2. Identify ambiguities and gaps
3. Ask clarifying questions
4. Refine requirements
5. Document decisions and trade-offs

## Output
Update `.specify/specs/<feature-name>/spec.md` with:
- Clarified requirements
- Resolved ambiguities
- Documented trade-offs
- Updated acceptance criteria
