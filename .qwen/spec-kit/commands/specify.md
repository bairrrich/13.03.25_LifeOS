# /speckit.specify

## Purpose
Define functional requirements for a new feature.

## Instructions
1. Understand the feature goal
2. Identify user stories and use cases
3. Define functional requirements
4. Document edge cases and constraints
5. Specify acceptance criteria

## Output
Create `.specify/specs/<feature-name>/spec.md` with:
- Overview and goals
- User stories
- Functional requirements
- Non-functional requirements
- Acceptance criteria
- Open questions
